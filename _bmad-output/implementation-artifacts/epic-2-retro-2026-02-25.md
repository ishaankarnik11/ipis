## Successes & Strengths

- **Cleanup-first pattern proven** — Stories 2.0/2.0b resolved all Epic 1 debt and established a 368-test baseline before feature work. This pattern is now institutionalized.
- **Row-level bulk upload validation** — Non-atomic processing was the right architectural choice. Valid rows import regardless of invalid rows. Pattern reusable for Epic 5 uploads.
- **Reusable components built for Epic 5** — UploadConfirmationCard, UploadHistoryLog, `postForm()` helper designed for reuse in timesheet/billing uploads.
- **RBAC field filtering** — Sophisticated access pattern: HR manages employee records without seeing salary data, Finance sees everything. Clean implementation in service layer.
- **Strong Epic 1 retro follow-through** — 7/8 action items completed. Documentation created and referenced.
- **Zero blockers, zero incidents** — 100% story completion with no production issues.

## Challenges & Growth Areas

- **Code review finding volume still too high** — ~4 findings per story average, including 3 HIGH severity across the epic. Dell flagged this as a primary concern.
- **~50% of findings trace to AC quality gaps** — ACs covered happy paths well but missed: negative paths, concurrency behavior, input validation boundaries, error UX, security implications.
- **~50% of findings are dev execution gaps** — Edge-case tests missing, type safety erosion (`Record<string, unknown>`), antd component type mismatches.
- **Pre-review AC checklist never became a formal gate** — Despite being an Epic 1 commitment, it was partially implemented but not enforced.
- **`docs/gotchas.md` not keeping pace** — antd v6 `Spin.tip` → `description` in Story 2.4 not in gotchas doc despite the doc existing.
- **No E2E testing** — All frontend tests are component-level with mocked APIs. Dell had not seen a working product demo after Epic 2 completion.

## Key Insights & Learnings

1. **Better ACs eliminate roughly half of all code review findings** — A Defensive AC Checklist addressing negative paths, concurrency, input boundaries, error UX, and security would have prevented ~50% of findings.
2. **Documenting a fix is not the same as enforcing it** — The Epic 1 AC checklist action item was documented but not structural. Process changes must be mandatory workflow steps, not optional references.
3. **The cleanup-first story pattern works and should be institutionalized** — Every epic should start with a Story X.0 addressing previous retro actionables + regression verification.
4. **Mocked tests aren't enough** — 391 passing tests with mocked APIs don't prove the full system works. Playwright E2E tests close the gap between unit tests and real user experience.
