## Key Successes

1. **Pure Function TDD Architecture** — Stories 4-1 through 4-4 used pure functions with zero I/O. TDD'd against Dell's Excel reference fixtures. Zero debug issues on calculator stories. Dramatically faster and cleaner than integration-heavy stories.

2. **Data Contract Table Pattern Applied** — Epic 3 retro action item paid off. Story 4-0 traced all 5 fields (slaDescription, vendorCostPaise, manpowerCostPaise, budgetPaise, infraCostMode) from UI→Zod→Prisma→E2E. Gate caught H1 finding.

3. **100% Story Completion, Zero Production Incidents** — All 7 stories delivered with all acceptance criteria met. No production issues.

4. **E2E Test Suite Growth** — From 51 tests at epic start to 67 by end. Cross-role chains (Story 4-0b) validate complete user journeys across 5 roles.

5. **EmployeeCostEntry Type Unification** — Story 4-4 renamed `TmEmployeeCost` → `EmployeeCostEntry` with backward-compat alias. Clean pattern sharing across T&M, Fixed Cost, AMC, and Infrastructure DETAILED models.

6. **Discriminated Union Pattern** — Infrastructure SIMPLE vs DETAILED with exhaustive `never` check became a reference pattern for type-safe model discrimination.

7. **Epic 3 Retro Follow-Through** — 8/10 action items completed. Structural workflow gates (Data Contract table, Persist-and-Verify, HIGH Findings Gate) all caught real issues.

## Critical Challenges

1. **Story 4-5 False Completion Claim** — Review 1 M3 claimed BreakdownJson was replaced with a proper union type, but code still had `Record<string, unknown>`. Required a second review round to catch. 16 total findings across 2 reviews.

2. **Pre-existing Test Failures Normalized** — Failing tests carried forward from story to story and from previous epics. "Inherited, not related to this story" became an accepted excuse. Failures span backend unit, frontend unit, and E2E suites.

3. **Master Test Plan Sync Drift** — Stories 4-3, 4-4, and 4-5 (3/7 stories) had tests written but `master-test-plan.md` still showing stale status (DEVELOPED_UNTESTED or NOT_DEVELOPED). Manual sync process is error-prone.

4. **Type Safety Gaps at Boundaries** — Story 4-3 H1 (nullable completionPercent typed as non-nullable), Story 4-5 H1 (loose BreakdownJson type), Story 4-0 M7 (unsafe TypeScript cast). All caught only in review, not during development.

5. **Unit Test Gaps from 4-0b Never Resolved** — Three MEDIUM review follow-ups documented but never completed: AddTeamMemberModal tests, API function tests, canManageTeam guard test.

6. **Snapshot Engine Not Wired** — Story 4-5 Task 5 (wire `persistSnapshots()` into `upload.service.ts`) explicitly deferred to Epic 5. Engine built and tested but not connected.

## Team Agreements (Epic 4)

1. **"Structural gates, not team agreements"** — Every process improvement must be embedded in workflow files. If it can't be a gate, it's aspirational, not enforceable.
2. **"Inherited failures are still failures"** — The test suite must be 100% green at all times. No story moves to `review` with any failing tests, regardless of origin.
3. **"Verify the fix, not the claim"** — Review verification checks actual code diffs, not just the developer's assertion that a finding was fixed.
4. **"Pure functions where possible"** — Isolate business logic from I/O. TDD pure functions for fast, reliable development. Apply the calculator pattern wherever applicable.

## Key Takeaways

4. **Verify the fix, not the claim** — Story 4-5's false completion claim shows review verification needs code-level checking, not trust.
