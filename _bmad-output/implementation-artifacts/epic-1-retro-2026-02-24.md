## Successes & Strengths

- **Rock-solid foundation** — Monorepo scaffold (Story 1.1) established consistent patterns that all subsequent stories built on cleanly
- **Consistent architecture discipline** — Layer separation (routes → services → Prisma) and middleware chain pattern (auth → RBAC → validate → asyncHandler) maintained across all 6 stories
- **Security-first approach** — Timing-safe bcrypt (1.2), SHA-256 token hashing (1.6), TOCTOU fix via interactive Prisma transaction (1.6), user enumeration prevention in login and forgot-password flows
- **Impressive test growth** — 12 to 224 tests with TDD approach followed, covering unit, integration, and component layers
- **Code review process** — Caught 11 HIGH severity issues including login race condition (1.3), TOCTOU race in password reset (1.6), missing AC functionality (1.5)
- **Story-to-story knowledge transfer** — "Previous Story Intelligence" sections in each story prevented duplicate work and maintained pattern consistency

## Challenges & Growth Areas

- **antd v6 API differences repeated across 3 stories** — Alert `message` → `title` prop rename hit Stories 1.3, 1.5, and 1.6 independently. `destroyOnClose` → `destroyOnHidden` hit Story 1.5. Knowledge was documented in change logs but not referenced by later stories.
- **Wrong dependency versions in initial scaffold** — Story 1.1 code review caught antd pinned to v5 (should be v6.3.0), Vite to v6 (should be v7.3.1), jose to v6 (should be v5). All caught in review but avoidable.
- **Windows testing infrastructure pain** — vitest `pool: 'forks'` causes `spawn UNKNOWN` (1.3), jsdom lacks `window.matchMedia` (1.3), antd CSS transitions break test rendering (1.5), extended test timeouts needed (1.5)
- **High code review finding density** — ~2 HIGH issues per story average. Stories 1.3 and 1.6 required two review rounds each.
- **AC violations slipping to review** — Story 1.5 missing `message.success` toast and missing `departmentName` in User interface — basic requirement coverage gaps found only in code review.

## Key Insights & Learnings

1. **Knowledge must live in canonical docs, not buried in story change logs** — Same antd v6 mistake repeated 3 times because fixes were documented per-story but not shared globally
2. **Pre-review AC verification prevents an entire category of review findings** — A simple checklist before submitting for review would catch requirement coverage gaps
3. **Upfront architecture investment pays off** — Detailed story specs and consistent patterns kept issues minimal per story
4. **Code review is essential** — Caught critical security issues (race conditions, TOCTOU) that would have been hard to detect otherwise
