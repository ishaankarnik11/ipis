Six visual directions for the Internal Profitability Intelligence System. Each explores a different balance of density, navigation, and visual weight.

D1 — The Ledger
Maximum density. Pure tabular. Bloomberg-terminal energy. Tables-first, charts secondary.
Best for: Data-first analysts →
D2 — The Dashboard
KPI cards prominent at top. Charts below. Clean executive view with details accessible.
Best for: Leadership scanning →
D3 — Navy Command
Navy sidebar expands into a rich module panel. Dark-header, white content. High authority.
Best for: Power users / Finance →
D4 — The Analyst
Split view: summary metrics left panel, detail table right. Two-pane pattern for deep work.
Best for: Investigation workflows →
D5 — Card Grid
Entity-per-card grid view. Projects/employees as browsable cards with sparklines and badges.
Best for: Portfolio browsing →
D6 — Compact Pro
Icon-only sidebar. Maximum content width. Ultra-dense tables. Pro tool aesthetic.
Best for: Expert daily users →

What this direction does well

  • Maximum information density — Priya sees the most data per scroll
  • Full sidebar always visible — zero navigation ambiguity
  • Dotted-underline margin figures invite the Ledger Drawer naturally
  • Tables-first matches the P&L mental model

Trade-offs

  • 220px sidebar consumes screen width on smaller displays
  • No visual charts — leadership may prefer a graphical overview
  • Dense tables require user comfort with financial data
Recommended for: Priya (Finance) as her primary working view. The Ledger direction is built for the person who lives in this data daily.

What this direction does well

  • Navy header communicates authority and brand strength immediately
  • Top navigation bar is more familiar to Jira/Zoho users than a left sider
  • Left-border KPI cards are a clean financial report pattern
  • Full-width content area maximizes table visibility

Trade-offs

  • Top nav compresses navigation items — may not scale to 5+ modules
  • Two header bars (top nav + sub-header) use vertical space
  • Less navigation depth than sider-based approaches
Best element to borrow: The navy sub-header with inline period filter and breadcrumb. This is a strong pattern regardless of chosen direction.

What this direction does well

  • Summary panel always visible alongside detail table — no scrolling to get context
  • Alert indicators in summary panel (⚠ 1 project at loss) are immediately actionable
  • Loss-row highlighting draws the eye without a badge alone
  • Two-pane mirrors Jira's panel model — familiar to the team

Trade-offs

  • May feel redundant when the Ledger Drawer is already the investigation mechanism
Best element to borrow: The alert indicators in the summary panel ("⚠ 1 project at loss") and the loss-row background highlight. Both add signal without adding noise.

What this direction does well

  • Maximum content width — the most rows visible per screen of any direction
  • KPI strip across the top is space-efficient and scannable
  • "At Risk: 2" KPI tile in red immediately draws the eye to problems
  • Ultra-compact tables are ideal for Priya's daily P&L review

Trade-offs

  • Icon-only sider requires users to memorise icon meanings
  • Very small touch targets — not suitable for tablet/leadership access
  • Compact KPI strip loses the "delta vs target" context of bigger cards
Best element to borrow: The KPI strip with an "At Risk: 2" alert tile in red. Turning an aggregate alert into a KPI slot is a powerful pattern — it surfaces the problem count before the user reads the table.
